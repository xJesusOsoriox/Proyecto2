Direcciones IP

Cada dirección es única para cada una de las interfaces de red, por lo que, si un dispositivo dispone de más de una interfaz de red, necesitará una dirección IP para cada una












Las direcciones IP proporcionan dos datos: el número de red y el número de host. Las direcciones IP tienen una longitud de 32 bits o 4 bytes. De cada conversión resulta un número comprendido entre 0 y 255, los cuales se escriben separados entre sí por un punto.

Los sistemas de red se direccionan de 3 formas:
El protocolo IP distingue la clase a la que pertenece una dirección analizando el valor de sus bits de mayor peso. Para determinar la clase de una dirección IP se siguen las siguientes reglas:
Unicast
Los paquetes de datos tienen como destino la dirección de un único host.
Clase A
Si el bit de mayor peso es 0, la mascará tendrá 8 bits. Se tiene por tanto 8 bits para direcciones de red y 24 bits para host. Pensada para empresas o corporaciones con grandes terminales
De 0.0.0.0 a 127.255.255.255
Multcast
Los datos se pueden enviar de forma simultánea a un determinado conjunto de host.
Clase B
Si los dos primeros bits son 1 y 0, la mascará tendrá 16 bits. Se tiene por tanto 16 bits para direcciones de red y 16 bits para host. Pensada para corporaciones medianas.
De 128.0.0.0 a 191.255.255.255
Broadcast
Permite enviar datos a todos los sistemas que forman parte de una red.
Clase C
Si los tres primeros bits son 1, 1, y 0, la mascará tendrá 24 bits. Se tiene por tanto 24 bits para direcciones de red y 8 bits para host. Pensada para entornos pequeños.
De 192.0.0.0 a 223.255.255.255
